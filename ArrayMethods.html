Introduction
The primary purpose of this module is to provide common array manipulation functionality. The module was written for Visual Basic 6.0, but should be compatible with most other versions.

Like most typical programming languages, VB includes some built-in provisions for array manipulation. They can be used to create, resize, and copy whole arrays, to determine their sizes and to read and write individual items within. Unfortunately, not included are a number of routines for manipulating whole arrays. What if one wanted an array to be a copy of just the first twenty items of another array? To concatenate two arrays together? To insert one or more items into the middle of an array?

This module makes up for such inadequacies by providing a number of routines to perform more sophisticated tasks like these.

This module does not depend on any libraries beyond the one that comes standard with Visual Basic.

Applying this Module
Here is some help for the programmer on how to use this module in his own applications.

Most of the routines in this module are designed to move stuff around within or merge arrays. Most of those tasks can be achieved by using the most esoteric one: ArraySplice(). ArraySplice() essentially removes a single section of an array and inserts the contents of another array in its place. You could use this subroutine, for example, to remove items four through seven from an array and replace that section with the three items in another array. The following illustrates:

    ArraySplice ListA, 4, 3, ListB
    ListA initially:
    A B C D E F G H I J
    		     ListB:
    x y z
    ListA after removal:
    A B C D I J
    ListA after insertion:
    A B C D x y z I J
The section doesn't have to be in the middle. One can work at either end of the array. Further, it isn't necessary to either remove anything (Length = 0) or to insert anything afterwards.

While ArraySplice() can be used to do things like inserting single items (Using ListToInsert = Array(ItemToInsert)) or list concatenation, you may find many of the other routines are simpler, easier to read, and more efficiently coded, because of their specialization.

Public Methods
This module offers the following publicly available functions and subroutines:

ArrayAppend | ArrayConcatenate | ArrayCopy | ArrayInsert | ArrayRemove | ArraySplice | ArrayToString | ArrayTrim

ArrayAppend
ArrayAppend(List, Item)

Add a new item to the end of the array.

ArrayConcatenate
Sub ArrayConcatenate(List, ListToCopyFrom)

Concatenate (copy) one list's contents to the end of another.

ArrayCopy
Sub ArrayCopy(List, ListToCopyFrom, Optional ByVal StartAt As Integer = 0, Optional ByVal Length As Integer = -1)

Copy the first array's contents to the second. Allows a subset of the array to be specified with a start point and length. Please note that if there are no items to copy, the array may still end up with one item, since not all arrays can have zero items. A Length value less than zero is the same as not specifying a length: all items after StartAt will be copied.

ArrayInsert
Sub ArrayInsert(List, ByVal Index As Integer, Item)

Create a gap in the array before the specified index and insert the item into it.

ArrayRemove
Sub ArrayRemove(List, ByVal Index As Integer)

Remove an item from the array and close the gap with whatever was above it. Note that while a Variant with an array can have zero items, other kinds of array require at least one item. This means that if you pass a one-item array that can't be shrunk down to zero, the last item will be left alone.

ArraySplice
Sub ArraySplice(List, ByVal StartAt As Long, ByVal Length As Long, Optional ListToInsert)

Cut a section out of the list and (optionally) insert the contents of another list. For example:

    ArraySplice
    A B C D E
    		, 2, 2,
    x y z
yeilds:
A B x y z E

See Applying This Module for more on the utility of this subroutine.

ArrayToString
Function ArrayToString(List, Optional ByVal Separator As String = vbCrLf) As String

Create a string representation of the array. If you don't specify a separator, , the Windows standard new-line combination, is assumed. The following values have special representations:

    Null {Null}
    Empty {Empty}
    Object Ref {Object:<type>}
    (e.g., {Object:Collection} )

This routine is great for debugging apps that use arrays.

ArrayTrim
Sub ArrayTrim(List, Optional ByVal TrimFromBeginning As Integer = 0, Optional ByVal TrimFromEnd As Integer = 0)

Trim items from the beginning and/or end of the array. For example:

    ArrayTrim
    A B C D E
    		, 2, 1
yeilds:
C D
